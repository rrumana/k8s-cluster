### Harbor Placement Policy

- `apps-private`
  - First-party images only.
  - Current examples: Portfolio prod/staging.

- `proxy-*`
  - Use for low-blast-radius, high-churn, or easy-to-rebuild third-party images.
  - This is the right home for experimental and convenience services.
  - Recommended examples:
    - LibreChat
    - Hypermind
    - Whiteboard
    - Collabora
    - Uptime Kuma
    - ARR sidecars and similar utility images

- `mirror`
  - Use for third-party images that should be copied deliberately and only changed during planned maintenance windows.
  - This is the right home for large-blast-radius platform components and core user-facing services.
  - Recommended examples:
    - Rook/Ceph
    - Linkerd
    - HAProxy Ingress
    - CNPG / Postgres-related runtime images
    - Valkey
    - cert-manager
    - External Secrets
    - VolSync
    - Harbor
    - Vault
    - Nextcloud
    - Immich
    - Jellyfin
    - Headscale
    - Vaultwarden

- `thirdparty-charts`
  - Mirror Helm OCI charts that Argo depends on.
  - This is the next resilience step after runtime image migration.
